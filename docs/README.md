# Documentation

This folder holds the core documentation for the training environment.

## Contents

- [daily plan](daily-plan.md): a suggested structure for each two-hour learning session
- future docs can include architecture notes, prompt design guidelines, and evaluation rubrics

## Suggested Use

Use these documents as reference points when shaping the daily experience for faculty members and when extending the platform with new features.

## Participant Workflow

When multiple faculty members use this repository:

- each person should fork and clone the repository
- each day they should work in their own branch
- each day's summary should be saved in the summaries folder using the naming convention <name-date>.md
- the final step each day is to open a pull request with their progress and summary
